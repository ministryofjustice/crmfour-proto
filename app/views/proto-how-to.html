Rebrand

The overall proto has been updated to use the new branding. The older versions of the service use a different layout template: main-old.html where the branding is switched off.

Versioning

When using routes for conditional questions, like radio's buttons, the form action needs to have {{version}} included so it will use the correct version as specfied in the page:

Variables

Variables are used in this prototype to store, remember and display data on the page. They can be seen in the URL and look like this: AppStatus=Rejected.

The ampersand '&' symbol can be used to seperate and apply multiple variables to a page, for example: cwAppStatus=Rejected&Appeal=Yes.

Variables used in this prototype include:

cwAppStatus

Used to define the current state of a claim for caseworkers.

  • cwAppStatus=New
  • cwAppStatus=In progress
  • cwAppStatus=Part granted
  • cwAppStatus=Granted
  • cwAppStatus=Rejected
  • cwAppStatus=Expired
  • cwAppStatus=Hold
  • cwAppStatus=Referred
  • cwAppStatus=Review
  • cwAppStatus=Returned

Appeal

Used to display an appeal on the claim.

  • Appeal=Yes
  • Appeal=No
